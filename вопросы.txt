1.Для чего служат формы на веб-страницах? 
Для получения информации от пользователя. Все логины, пароли, выпадающие спики, комментарии и тд.

2.Каким типом задаётся поле ввода в форме текста?
с помощью элемента <input type="text"> 

3.Какими способами можно обратиться к элементу формы?
Можно по индексу, а можно по имени

4.что такое событие submit для формы и как его можно использовать?
Событие submit нужно для отпрвки данных, оно собирает данные из всех полей ввода, отправляет на адсес указанный в action, потом перезагружает страницу.
Можно сделать так чтобы не перезагружал и обработать данные как-нибудь на свое усмотрение.

5.Восстановить значения умолчания в полях формы можно только по кнопке reset?
Везде пишут что вроде это основной способ. Но например если отсутствует атрибут action, то страница перезагрузится и восстановит значения по умолчанию. Наверное можно еще вручную стереть.

6.Каким способом можно отправить данные из формы на сервер?
при помощи фтрибута action <form action="путь_к_серверному_файлу">

7.Как можно полностью перехватить и обработать js-функцией событие submit (отправку данных на сервер)?
пишется сначала event.preventDefault();
Потом свой код для обработки данных из форм.

8.Какие есть недостатки у стандартного способа задания валидации через HTML5?
То что его довольно просто обойти, если надо - валидация по разному работает в некоторых браузерах, к тому же для того чтобы она корректно работала должен быть включен java script. Ваидация через HTML5 умеет не всё - какие-то сложные вещи все равно надо будет писать вручную.

9.Какие свойства доступны в объекте validity для проверки валидности элемента формы?
valid: если элемент без ошибок возвращает true
valueMissing: проверяет пустое ли поле
typeMismatch: проверяет правильно заполнено поле или нет
tooLong проверяет не длиннеели заполенние поля чем максимально допустимая длина
tooShort не короче чем минильно допустимая
customError - в зависимости от того установлено ли сообщение об ошибке возвратит true или false 
stepMismatch -  это я не поняла
patternMismatch - сообщение не соответствует тому что находится в атрибуте паттерн
rangeOverflow возвращает true или false в зависимости от того, превышает или нет значение верхний диапазон атрибута max 
rangeUnderflow - наоборот, меньше или нет атрибута min 

10.как можно проверить, является ли элемент формы валидным с использованием объекта validity?
создать функцию и прогнать по всем необходимым свойствам.

11.Какие флаги могут быть использованы при создании регулярных выражений и какова их роль?
i - делает поиск независимым от регистра
g - глабальный поиск, то есть все совпадания
y - поиск на конкретной позиции в тексте
m - многострочный режим

12.Что делает метод match() при работе с регулярными выражениями? Какие аргументы он принимает?
В качестве аргумента он принимает регулярное выражение. И возвращает совпадения в виде массива.

13. Что делает метод `test()` при проверке регулярного выражения на совпадение? Какие значения он возвращает?
Он ищет хотя бы одно совпадение в строке. Возвращает булевые значения
